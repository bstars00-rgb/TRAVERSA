## 2. 미해결 이슈와 리스크

### 기술
- **번들 크기**: `charts`(recharts) 청크가 gzip 92KB. 현재 수동 청크 분할로 완화했으나 FutureJourneyPage에서만 사용 → 추후 lazy import 검토 여지.
- **Mock 한계**: 모든 데이터가 시드/규칙 기반. 실제 매핑 정확도·정책 파싱·가격 스냅샷 괴리 문제는 실서비스 전환 시 발생 (docs 문서 §8 기술 리스크 참고).
- **줄바꿈**: `text-wrap: pretty/balance`는 최신 브라우저 기능 → 구형 브라우저에서 미적용(기능 저하 없음, 미관만).

### 사업 (검증 필요 가설 — docs 문서 §8)
1. 사용자가 필터 UI보다 대화형 설계를 더 신뢰하고 완주하는가 (전환율)
2. "3전략 압축"이 수백 개 리스트보다 높은 예약 전환을 만드는가
3. 위험 경고 노출이 신뢰를 높이는가, 이탈을 만드는가
4. 공급사가 B2B 계약으로 재고·요금을 제공할 유인이 있는가
5. Human-in-the-loop 승인 단계가 이탈 지점이 되지 않는 최소 마찰 수준
